# Tabimap - Development Log (July 21, 2026)

## 1. User Profile & Preferences Modals (UI)
- **Account Dropdown**: Added an interactive Avatar dropdown menu to the top navigation for authenticated users containing options for "Edit Profile", "Set Preferences", and a prominent red "Sign Out" button.
- **Preferences Modal**: Designed and built a `Set Preferences` modal UI to let users easily customize their trip pacing and transportation preferences.
- **Stacking Context Fixes**: Resolved a critical z-index and `backdrop-filter` rendering issue in Safari/Chrome by utilizing React's `createPortal`, ensuring modals overlay the entire app seamlessly without background bleed.
- **Mobile Responsiveness**: Upgraded the mobile hamburger menu to include all authentication controls and refactored modals to be fully scrollable on extremely small device screens.

## 2. Legal Pages & Semantic Versioning
- **App Versioning**: Implemented an automated application versioning system (`v1.3.0`). Configured Vite's `define` build options to automatically pull the version from `package.json` into a globally accessible `__APP_VERSION__` variable for type-safe rendering.
- **Legal Infrastructure**: Developed `Terms.tsx`, `Privacy.tsx`, and `Cookies.tsx` pages in a dedicated `features/legal/` module.
- **Global Footer**: Added a sticky global `Footer.tsx` that proudly displays the current app version alongside dynamic links to the new legal pages.

## 3. Cloudflare Deployment Pipeline & CI Repairs
- **Worker Configuration Conflict**: Investigated a total CI pipeline failure (the `Workers Builds: trip-planner` check was constantly failing). Discovered the root cause was the repository being incorrectly treated as a Cloudflare Worker due to an obsolete `wrangler.toml` file containing old D1 database bindings.
- **Cleanup**: Deleted `wrangler.toml` and removed the dead `functions/` directory, cementing the architecture as a pure frontend Cloudflare Pages app backed by Supabase.
- **Vite 8 Rolldown Fix**: Resolved an esoteric `npm ci` bug where Cloudflare Pages could not download Vite's native Linux `rolldown` bindings (`@rolldown/binding-linux-x64-gnu`). Fixed this by deleting `package-lock.json` and updating GitHub Actions to run standard `npm install`.
- **Node.js Environment Sync**: Updated `.node-version` and `.github/workflows/ci.yml` strictly to Node.js v22 to perfectly align with Vite 8's strict engine requirements. 
- Successfully deployed the `main` branch to the production `tabimap` Pages project.

## 4. Advanced Security Hardening (MDN 100/100 A+)
- **CSP Lockdown**: Analyzed the Mozilla MDN HTTP Observatory report and removed the `-20` penalty by eliminating `'unsafe-inline'` from the `script-src` Content-Security-Policy (CSP) directive.
- **Advanced Policies**: Further secured the CSP by locking down `object-src 'none'`, `base-uri 'none'`, and `form-action 'self'`.
- **Bonus Security Headers**: Added `Strict-Transport-Security` (HSTS) with `preload`, `Cross-Origin-Resource-Policy` (CORP), and `Cross-Origin-Opener-Policy` (COOP) to `public/_headers`.
- **Result**: Achieved the maximum possible score (125/100, A+) on the HTTP Observatory.
