# Tabimap - Development Log (July 20, 2026)

## 1. Authentication Migration (Clerk -> Supabase)
- **Supabase Integration**: Created `supabase.ts` client to connect to the Supabase backend.
- **Custom Auth Hook**: Developed `useAuth.ts` hook exposing sign-in methods for Google, X/Twitter, LINE, and Email/Password.
- **Glassmorphism Auth Modal**: Designed and built `AuthModal.tsx`, a premium, animated glassmorphism login modal integrating the new Supabase auth methods.
- **UI Integration**: Swapped out Clerk's `SignInButton` and `UserButton` with custom implementation using `useAuth` hook and `AuthModal`.

## 2. Dynamic Distance Calculation Fix
- **Issue**: Travel estimates were falling back to default values or pointing to the wrong closest stations when users provided a Zip Code instead of a station.
- **Fix**: Implemented Nominatim API geocoding in `StationInput.tsx` to resolve raw Zip Codes into precise `lat`/`lng` coordinates.
- **State Management**: Persisted `homeStationCoords` in `useTripStore` so that the `BudgetService` and recommendation engine can dynamically compute distances (using the Haversine formula) regardless of whether a user picks a station or enters a zip code.

## 3. Prefecture Checklist Map
- **Feature**: Developed a visual, interactive SVG map of Japan to allow users to track the prefectures they've visited.
- **Design**: Implemented using `@react-map/japan` with custom coloring, hover states, and smooth tooltips. 

## 4. Codebase Refactoring & Ponytail Audit
- Ran a complete repository audit looking for over-engineering (via `/ponytail:ponytail-audit`).
- Identified and reported ~75 lines of redundant/dead code:
  - `<delete>` Unused `exportData` & `importData` functions in `useTripStore`.
  - `<delete>` Unused `compareDestinations` function in `DestinationService.ts`.
  - `<delete>` Unused `APP_CONFIG` object.
  - `<native>` Identified that the Haversine formula in `distance.ts` was reinventing the wheel since `Leaflet.js` (`L.latLng().distanceTo()`) is already installed.
  - `<shrink>` Identified object aliases (`destinationService`, `budgetService`) that just wrapped named exports.
  - `<yagni>` Identified unnecessary function re-exports in `utils.ts`.

## 5. Geographical Data Cleansing (Station Filters)
- **Issue**: The prefecture filter in the UI was showing incorrect stations (e.g., Ueno Station in Ibaraki instead of Tokyo). This occurred because the static JSON mapped stations by railway lines (e.g., Joban Line crosses Tokyo, Chiba, and Ibaraki), causing stations to bleed across borders.
- **Fix**: Wrote a Node.js data-processing script utilizing Turf.js (`@turf/boolean-point-in-polygon`) and a high-resolution GeoJSON map of Japanese prefectures. 
- **Result**: Successfully reverse-geocoded and mapped all **8,532** unique stations to their strictly correct geographical prefectures using their exact coordinates. Overwrote `stations-by-prefecture.json` and deployed.

## 6. Budget Calibration & Branding Alignment
- **Budget Refinement**: Calibrated realistic travel costs across trains, Shinkansen, highway buses, and rental cars (assuming a 2-person split). This ensures the `BudgetService` provides highly accurate estimates for users.
- **Branding**: Enforced the "TabiMap" branding consistently across the Footer, AuthModal, and documentation, retiring old placeholders like "Japan Trip Planner".
